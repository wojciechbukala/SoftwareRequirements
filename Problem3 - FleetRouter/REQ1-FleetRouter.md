Implement a program to support daily route planning for a courier company, called FleetRouter. The program reads all input data from CSV files, assigns packages to vehicles, computes the best route for each vehicle, and writes the results to several CSV output files.

The program takes four input files: *packages.csv* -  containing a unique package ID, a delivery location, weight in kg, volume in m³, a time window defined by opening and closing times, service duration, and a priority flag; *vehicles.csv* -containing a unique vehicle ID, maximum weight capacity in kg, maximum volume capacity in m³, and a depot location; *locations.csv* - containing a unique location ID and a name; *distances.csv* -  containing the distance in km and travel time in minutes for each pair of locations.

Generated output should consist of three files: stops_order.csv — containing a route ID, vehicle ID, stop order in the route, location ID, package ID, arrival time, and departure time; *undeliverable.csv* - containing the package ID and the reason it could not be delivered; *summary.csv* - containing the vehicle ID, total route distance in km, total route duration in minutes, and number of packages delivered.

Before planning begins, the program validates input data: packages or vehicles that reference a location ID not present in locations.csv shall be reported and excluded from processing, packages whose time window is invalid shall be recorded in undeliverable.csv with reason TIME_WINDOW, and a missing distance entry for any ordered pair of locations required by the plan shall be treated as an unreachable connection, with the affected package recorded in undeliverable.csv with reason UNREACHABLE.

Every vehicle starts and ends its route at its assigned depot and makes exactly one trip per day, departing from the depot at 08:00. Every driver in the company works for 8 hours a day. A package's delivery must be done within the specified time window, so if the vehicle arrives before the opening of the window - it must wait. If a package cannot be delivered without violating any of the constraints, it is added to the undeliverable list with exactly one reason code. The reason code shall identify the cause of rejection: CAPACITY_WEIGHT when no vehicle has enough remaining weight capacity; CAPACITY_VOLUME when no vehicle has enough remaining volume capacity; TIME_WINDOW when the earliest feasible arrival exceeds the package's time window closing time, or when the package's time window is itself invalid (opening time not strictly earlier than closing time); MAX_DRIVER_TIME when assigning the package would cause a driver's total route duration to exceed 8 hours; UNREACHABLE when the package's destination cannot be reached from any depot, or cannot be returned to its depot, due to a missing distance entry; and NO_VEHICLE when none of the above applies but no vehicle can accommodate the package within the final plan. Each undeliverable package shall appear exactly once in undeliverable.csv. Priority packages must be assigned before non-priority ones whenever capacity forces a choice. The program must attempt to deliver every package and optimise routes by minimising total distance first, then total duration.

All time values in input and output files shall use the format HH:MM; distances shall be expressed in kilometers rounded to two decimal places; and all durations shall be expressed as integer minutes. The program is invoked from the command line as fleetrouter --input <dir> --output <dir>, where both arguments are mandatory and specify, respectively, the directory containing the input files and the directory where output files are written.
