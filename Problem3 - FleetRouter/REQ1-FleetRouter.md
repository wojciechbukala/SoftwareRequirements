Implement a program to support daily route planning for a courier company, called FleetRouter. The program reads all input data from CSV files, assigns packages to vehicles, computes the best route for each vehicle, and writes the results to several CSV output files.

The program takes four input files: *packages.csv* -  containing a unique package ID, a delivery location, weight in kg, volume in m³, a time window defined by opening and closing times, service duration, and a priority flag; *vehicles.csv* -containing a unique vehicle ID, maximum weight capacity in kg, maximum volume capacity in m³, and a depot location; *locations.csv* - containing a unique location ID and a name; *distances.csv* -  containing the distance in km and travel time in minutes for each pair of locations.

Generated output should consists of three files: *routes.csv* -  containing the vehicle ID, stop order in the route, location ID, package ID, arrival time, and departure time; *undeliverable.csv* - containing the package ID and the reason it could not be delivered; *summary.csv* - containing the vehicle ID, total route distance in km, total route duration in minutes, number of packages delivered, and number of packages undelivered.

Every vahicle starts and ends its route at its assigned depot and makes exactly one trip per day. At all times, each vehicle must respect its weight and volume capacity limits. Every driver in the company works for 8 hours a day. A package's delivery must be done within the specified time window, so if the vehicle arrives before the opening of the window - it must wait. If a package cannot be delivered without violating any of the constraints, it is added to the undeliverable list along with the reason. Priority packages must be assigned before non-priority ones whenever capacity forces a choice. The program must attempt to deliver every package and optimise routes by minimising total distance first, then total duration.
